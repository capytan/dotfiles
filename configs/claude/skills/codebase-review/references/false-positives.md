# False Positive Patterns (Codebase Review)

Shared catalog for all reviewers and scoring agents. Do not report findings matching these patterns.

## Tooling Catches

- **Linter / formatter detectable**: ESLint, Prettier, Ruff, Clippy, etc.
- **Type checker detectable**: TypeScript, mypy, Flow, etc.
- **Compiler detectable**: Syntax errors, unused imports, build-time errors
- **Test detectable**: Bugs caught by existing test suites

## Intentional Patterns

- **Intentional TODO / HACK**: Temporary workarounds with explanatory comments
- **lint-ignore / type-ignore**: Explicitly suppressed warnings
- **Intentional type casts**: `as any`, `# type: ignore` with explicit override
- **Feature flags**: Code behind feature flags that appears unused

## Framework & Library Patterns

- **Framework-idiomatic code**: Patterns recommended by the framework (React hooks, Django ORM, etc.)
- **Generated code**: Files auto-generated by tools (protobuf, OpenAPI, GraphQL codegen)
- **Boilerplate**: Framework-required boilerplate (config files, entry points)
- **Convention-over-configuration**: Framework defaults that look like missing config

## Test Code

- **Test fixtures and mock data**: Intentionally abnormal values
- **Test utilities**: Helper functions used only in tests
- **Test-only patterns**: Mocks, stubs, spies that bypass normal validation

## Vendor & Third-Party

- **Vendor directories**: Code in vendor/, node_modules/, third_party/
- **Copied library code**: Vendored code with attribution comments
- **Lock files**: Auto-generated dependency lock files

## Style & Convention

- **Pedantic style**: Naming preferences, formatting choices (unless explicit in CLAUDE.md)
- **Missing comments**: Lack of documentation (unless required by CLAUDE.md)
- **Missing tests**: Insufficient test coverage (unless required by CLAUDE.md)
- **General quality**: "Could be written better" level observations

## Low-Signal Patterns

- **"Might break" speculation**: No realistic path delivers the failure
- **Theoretical vulnerabilities**: Security issues with no attack surface (internal tools, local-only)
- **Performance speculation**: "Might be slow" without benchmarks
- **Unused code that's clearly staged**: Recently added code awaiting integration

## Reviewer-Specific Exceptions

### Architecture Analyst
- Framework-mandated directory structure
- Test file organization separate from source
- Build/config files in project root

### Security Scanner
- Excessive validation demands on internal-only APIs
- Security concerns on dev/test-only code
- Example/demo credentials clearly marked as non-production

### Dependency Analyst
- Dev-only dependencies with known vulnerabilities (not in production bundle)
- Pinned versions with documented reasons
- Workspace-level dependency hoisting
