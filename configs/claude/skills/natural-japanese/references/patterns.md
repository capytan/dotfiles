## A. 語彙レベル

### 「これにより」の置き換え

人間の文章ではほぼ使われないのに、生成AIの文章では異常に頻出する。

**例1:**
- AI: 「新しい管理システムを導入しました。これにより、業務効率が30%向上しました」
- 自然: 「新しい管理システムを導入した結果、業務効率が30%上がった」

**例2:**
- AI: 「研修プログラムを刷新しました。これにより、社員の満足度が改善しました」
- 自然: 「研修プログラムを刷新したところ、社員の満足度が上がった」

**例3（削除パターン）:**
- AI: 「APIを統合しました。これにより、データの一元管理が可能になりました」
- 自然: 「APIを統合して、データを一元管理できるようにした」

### 「〜することができます」→「〜できます」

**例:**
- AI: 「このツールを使用することで、作業時間を短縮することができます」
- 自然: 「このツールを使えば、作業時間を短縮できます」

### 抽象動詞の具体化

| AI的表現 | 自然な表現の例 |
|---|---|
| 「サポートする」 | 手伝う、助ける、支える、面倒を見る |
| 「提供する」 | 届ける、渡す、出す、用意する |
| 「活用する」 | 使う、取り入れる、役立てる |
| 「実現する」 | 作る、始める、叶える、形にする |
| 「確保する」 | 取る、押さえる、用意する |
| 「促進する」 | 進める、後押しする、広げる |
| 「最適化する」 | 調整する、整える、合わせる |
| 「強化する」 | 鍛える、補う、厚くする |
| 「構築する」 | 作る、組み立てる、立ち上げる |
| 「展開する」 | 広げる、始める、進める |

**例:**
- AI: 「弊社はお客様のビジネスをサポートし、最適なソリューションを提供します」
- 自然: 「お客様の業務に合った仕組みを一緒に作ります」

### 抽象形容詞の具体化

| AI的表現 | 変換方針 |
|---|---|
| 「本質的な」 | 何が本質なのか具体的に書く。書けないなら削除 |
| 「革新的な」 | 何がどう新しいのか書く |
| 「効果的な」 | 何にどう効くのか書く |
| 「重要な」 | なぜ重要かを書くか、削除して直接述べる |
| 「最適な」 | 何に対して最適かを書く |
| 「包括的な」 | 何を含んでいるかを列挙する |
| 「戦略的な」 | 具体的にどんな狙いか書く |

### 意義の過剰強調

**AI:**
> この取り組みは、業界全体のDX推進において極めて重要な役割を果たしており、その意義は計り知れない。

**自然:**
> この取り組みで、社内の申請処理が3日から4時間に縮まった。

「重要な」「極めて」「計り知れない」を消して、具体的な数字・事実に置き換える。

### 定型評価語

以下が出てきたらほぼ確実にAI生成。すべて書き換え対象:

| NGワード | 置き換え方 |
|---------|-----------|
| 浮き彫りにしており | 具体的に何が見えたか書く |
| 今後の展開が注目されます | 削除するか、自分の予測を書く |
| 多面的な / 包括的な / 画期的な | 何がどう多面的なのか具体化 |
| 注目に値する | なぜ自分が注目したか1文で |
| 重要な示唆を与えている | 何の示唆か具体的に |

## B. 構造レベル

### 語尾の単調さを直す

**AI（単調）:**
> リモートワークが普及しています。多くの企業が導入しています。
> 生産性の向上が報告されています。課題も指摘されています。

**自然（リズムあり）:**
> リモートワークが広がった。導入する企業は増える一方だ。
> 生産性が上がったという報告もあるが、課題がないわけではない。

語尾のバリエーション:
- 「〜だ」「〜である」（断定）
- 「〜だろう」「〜かもしれない」（推量）
- 「〜のだ」「〜わけだ」（説明）
- 体言止め（「〜という現実。」）
- 疑問形（「〜ではないだろうか」）
- 倒置（「難しい、この問題は」）

### 接続詞の過剰使用を直す

**AI（接続詞だらけ）:**
> まず、目標を設定します。次に、現状を分析します。
> さらに、課題を洗い出します。そして、解決策を検討します。
> 最後に、実行計画を立てます。

**自然:**
> 目標を決めたら、現状とのギャップを確認する。
> 課題が見えてきたところで解決策を考え、実行計画に落とし込む。

原則: 1文章に3回以上出たら多すぎ。接続詞は段落に1つあれば十分。文の配置と内容で流れを作る。

### 全角ダッシュ（——）

ChatGPT最大の書式癖。日本語でemダッシュで補足を挟むのは英語の直輸入。

**AI:**
> Claude Code——Anthropicが開発したCLIツール——を使えば

**自然:**
> Claude Code（AnthropicのCLIツール）を使えば

### 三点セット強制

AIはあらゆるものを3つにまとめたがる。メリット3つ、ステップ3つ、ポイント3つ。

**AI:**
> 3つのポイントにまとめると: 速度、安全性、コスト

**自然:**
> 速度と安全性は改善した。コストは微妙。あと運用のしやすさも良くなった。
> （4つ書く。無理に3つに丸めない）

### 否定並列構文

**AI:**
> 単なるツールではない。パラダイムシフトだ。
> コストの問題ではない。文化の問題だ。

問題: 「〜ではない。〜だ。」が連続するとAI感が強い。

**自然:**
> ただのツールじゃなくて、仕事のやり方自体が変わる。
> コストより文化の方が壁になる。

### 主語の過剰明示

日本語は主語を省略する言語。AIは英語の癖で主語を毎文入れる。

**AI:**
> このツールは高速です。このツールはセキュリティも強固です。このツールは無料で使えます。

**自然:**
> 高速で、セキュリティも強固。しかも無料。

### 見出しの過剰構造化

AIは短い文章でもh2/h3で細かく区切りたがる。

原則: 300字以下のセクションに小見出しは不要。読者が自然に読める流れを優先する。

## C. 意味レベル

### 安全クッション表現を削る

**AI:**
> 一般的に、プログラミング学習は継続が重要だと言われています。
> 多くの場合、初心者は3ヶ月程度で基礎を習得できると考えられています。

**自然:**
> プログラミングは続けないと身につかない。
> 基礎が身につくまでに、だいたい3ヶ月はかかる。

### 受動態・婉曲表現を能動態に

**AI:**
> この手法は多くの専門家によって推奨されています。
> 効果があると期待されています。

**自然:**
> 多くの専門家がこの手法を薦めている。
> 実際に効果が出ている。

### -ing的付け足し構文

データや事実の後に、AIが勝手に意義を付け足すパターン。

**AI:**
> 売上は前年比120%でした。これは同社の戦略が功を奏していることを示しており、今後の成長が期待されます。

**自然:**
> 売上は前年比120%だった。

読者に解釈を委ねる。「〜を示しており」「〜を物語っています」が続いたら丸ごと削除。

### 曖昧な出典

**AI:**
> 業界の専門家によると、多くの企業がこの手法を採用しています。

**自然:**
> Ahrefsが90万ページを分析したデータでは（具体的な出典）
> あるいは: 自分が見た範囲では、3社くらいがこのやり方をしてる（自分の観察として書く）

出典を出すなら具体的に。出せないなら「出典なし」と認めるか、自分の意見として書く。

## D. AI的な装飾

### 比喩の使いすぎを直す

**AI（比喩過多）:**
> データはビジネスの羅針盤です。AIはその推進力となるエンジンであり、
> 両者は成功への両輪と言えるでしょう。この土台の上に、
> イノベーションという架け橋を築いていく必要があります。

**自然:**
> データを見て判断し、AIで処理を自動化する。
> この2つが揃えば、新しいことを始める余裕が生まれる。

## E. 翻訳調

### 日本語圏の慣用と合わない表現

| AI的（翻訳調） | 自然な日本語 |
|---|---|
| 「あなたのビジネスを次のレベルへ」 | 具体的にどうなるか書く |
| 「ゲームチェンジャー」 | 「大きく変える」「転機になる」 |
| 「ジャーニー」（学習の旅） | 「道のり」「過程」 |
| 「インサイト」 | 「気づき」「発見」 |
| 「ベストプラクティス」 | 「うまくいくやり方」「定石」 |
| 「バリュー」 | 「価値」「良さ」 |
| 「リソース」 | 「人手」「予算」「時間」（具体的に） |
| 「ソリューション」 | 「解決策」「仕組み」「方法」 |
| 「イニシアチブ」 | 「取り組み」「施策」 |
| 「エコシステム」 | 「仕組み全体」「つながり」 |

### カタカナ語の連続

カタカナが3語以上連続したら疑う。

**AI:**
> ソリューションをレバレッジして、イノベーティブなアプローチでトランスフォーメーションを推進する。

**自然:**
> 既存の仕組みを活かして、新しいやり方で業務を変える。

## F. テンプレ感

### 前置き宣言を削る

**AI:**
> 本記事では、効果的なプレゼンテーションの方法について、
> 5つのポイントに分けて詳しく解説していきます。
> ぜひ最後までお読みください。

**自然:**
> プレゼンがうまくいくコツは5つある。

### 「いかがでしたか？」パターンを削る

**AI:**
> いかがでしたか？今回は〜について解説しました。
> 〜を活用して、ぜひ〜してみてください。

**自然:**
> （まとめの内容をそのまま書く。メタな振り返りは不要）

### 「ステップ」の強制構造化を崩す

**AI:**
> ステップ1: 目標を設定する
> ステップ2: 現状を把握する
> ステップ3: 計画を立てる

**自然（内容に応じて）:**
> まず目標を決める。今どこにいるかがわかれば、あとは計画を立てるだけだ。

番号付きステップが本当に必要な場面（手順書・チュートリアル）では残してよい。
ただし「STEP」という装飾語は不要で、普通の番号付きリストで十分。

### 追従的なトーン

**AI:**
> 素晴らしいご質問ですね！非常に良い指摘です。おっしゃる通りです。

**自然:**
> （削除して本題に入る。褒めるなら具体的に何が良いか書く）

読者に媚びない。

### 説教くさい前置き

**AI:**
> ここで重要なのは〜という点です。〜について理解しておく必要があります。注意すべき点として〜

**自然:**
> （前置きを消して、内容をそのまま書く）

読者は大人なので、何が重要かは自分で判断できる。

## G. Markdown構造

textlint-rule-preset-ai-writing の `no-ai-list-formatting`、`no-ai-emphasis-patterns`、
`no-ai-colon-continuation` に対応するパターン。

### 太字+コロンのリスト項目

**AI:**
> - **パフォーマンス**: レスポンスタイムを50%改善
> - **セキュリティ**: 脆弱性を自動検出
> - **スケーラビリティ**: 水平スケーリングに対応

**自然:**
> - パフォーマンス: レスポンスタイムを50%改善
> - セキュリティ: 脆弱性を自動検出
> - スケーラビリティ: 水平スケーリングに対応

太字を外す。ラベル+内容の形式自体は残してよいが、太字は不要。

### 絵文字プレフィックスのリスト

**AI:**
> ✅ テストが全て通過
> ❌ デプロイに失敗
> 💡 新しいアイデア

**自然:**
> - テストが全て通過
> - デプロイに失敗
> - 新しいアイデア

### 述語+コロン+ブロック要素

**AI:**
> 以下のコマンドを実行します:
> ```bash
> npm install
> ```

**自然:**
> 以下のコマンドを実行します。
> ```bash
> npm install
> ```

述語（動詞で終わる文）の後のコロンは句点に。名詞で終わる場合（「使用方法:」）は許容。

## H. 文体の統一

AI生成文が最もバレやすいポイントの一つが、文体の混在。
リライト作業自体が新たな混在を生まないよう注意が必要。

### です・ます調と、だ・である調の混在

**AI（混在）:**
> Gitのブランチ戦略、なんとなくで運用してない？
> ブランチ戦略を理解することは、チーム開発において不可欠です。
> 定番フローを押さえておこう。

**自然（だ・である調に統一）:**
> Gitのブランチ戦略、なんとなくで運用していないだろうか。
> ブランチ戦略を理解しておくことは、チーム開発で欠かせない。
> 定番フローを押さえておこう。

### カジュアル度の混在

**AI（混在）:**
> めっちゃ便利なツールを見つけました。
> 生産性の向上に大きく寄与するものと考えられます。
> ぜひ使ってみて！

問題: 「めっちゃ」「〜てみて！」（超カジュアル）と「寄与するものと考えられます」（論文調）が同居。

**自然（カジュアルに統一）:**
> めっちゃ便利なツールを見つけた。
> 生産性がかなり上がると思う。
> ぜひ使ってみて！

### 判断基準まとめ

| ユーザー指定 | 文体 | 語尾の例 |
|---|---|---|
| フォーマルビジネス | です・ます + 敬語 | 「〜いたします」「〜ございます」「〜でしょうか」 |
| 一般的なブログ | です・ます（ライト） | 「〜です」「〜ますね」「〜でしょう」 |
| テックブログ | だ・である | 「〜だ」「〜である」「〜してみた」「〜だろう」 |
| SNS/カジュアル | 口語 | 「〜だよ」「〜じゃん」「〜かも」 |

指定がない場合は、入力テキストの多数派の文体に合わせる。

## I. 声を入れる

### パターン除去だけでは空っぽになる

**AI（パターン除去後）:**
> AIツールの活用は今後重要になる。

問題: 定型語は消えたが、誰の文章でもない。声がない。

**自然（声を入れた後）:**
> 正直、もうAIなしで仕事するのは無理だと思ってる。

### 体験を入れる

**AI:**
> CI/CDの導入により、デプロイの効率が向上する。

**自然:**
> うちのチームは手動デプロイで月2回やらかしてた。CI/CD入れてからゼロになった。

### 不完全さを認める

**AI:**
> この手法は非常に効果的であり、すべてのプロジェクトに推奨できる。

**自然:**
> 自分のプロジェクトではうまくいった。ただ、大規模なやつで試したことはまだない。
